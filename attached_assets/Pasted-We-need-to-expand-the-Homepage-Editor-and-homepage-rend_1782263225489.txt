We need to expand the Homepage Editor and homepage rendering for the poster store project.

Important workflow constraints:

* Do not create branches.
* Do not commit or push to GitHub.
* Make the code changes only, then report changed files, tests/typecheck status, and any follow-up risks.
* Keep the implementation targeted and avoid broad refactors unless necessary.
* Preserve existing homepage behavior as fallback for existing store data.

Goal:
Create a flexible homepage editor where all homepage sections except the footer can be reordered, hidden/shown, and configured. Hero stays part of the homepage editor but footer is not included. Collection banners should support multiple instances.

Current known state:

* `stores.homepage_visual_config` already exists as JSONB.
* Current `HomepageVisualConfig` only supports `hero` and one `collectionBanner`.
* Current homepage renders fixed sections in this order: Hero, Featured posters, Collection banner, Explore region/category, New arrivals, Brand story, Value props.
* Current collection banner background uses a cropped cover image; this causes uploaded/designed banners to be cropped when the image aspect does not match the storefront aspect.
* Store Settings currently contains Theme colors and Brand & Typography, while Homepage Editor contains hero/collection visual settings. These should be harmonized so homepage design controls feel unified.

Required functionality:

1. Homepage sections model
   Extend `HomepageVisualConfig` to support an ordered section list below/including Hero, excluding footer.

Add a flexible structure like:

```ts
type HomepageSectionType =
  | "hero"
  | "featuredPosters"
  | "collectionBanner"
  | "exploreLinks"
  | "newArrivals"
  | "brandStory"
  | "valueProps";

interface HomepageSectionConfig {
  id: string;
  type: HomepageSectionType;
  visible: boolean;
  sortOrder: number;
  titleOverride?: string | null;
  fontOverrides?: SectionFontOverrides | null;
  colorOverrides?: SectionColorOverrides | null;
  bannerId?: string | null;
}

interface HomepageVisualConfig {
  hero?: HeroVisualConfig;
  sections?: HomepageSectionConfig[];
  collectionBanners?: CollectionBannerVisualConfig[];
  // Keep old collectionBanner field as backwards-compatible fallback.
  collectionBanner?: CollectionBannerVisualConfig;
}
```

Preserve backwards compatibility:

* If `sections` is missing, render current default section order.
* If `collectionBanners` is missing but old `collectionBanner` exists, render it as the first/default collection banner.
* Existing stores must not break.

2. Reorder all sections except footer
   In `AdminHomepageEditor`, add a “Homepage layout” area where sections can be:

* moved up/down
* hidden/shown
* reset/normalized to default order
* collection banners can be added, duplicated, removed, and moved
* all section rows/cards should show clear labels, e.g. “Hero”, “Featured posters”, “Collection banner: Mediterranean Walls”, “New arrivals”, etc.

Hero can be reorderable too unless it causes large layout complications. If it is safer to keep Hero locked at top in this first implementation, make that explicit in the report and keep all sections below Hero reorderable. But preferred end goal is that all homepage sections apart from footer are configurable.

3. Multiple collection banners
   Replace the current single collection banner editor with repeatable collection banner cards.

Each collection banner should support:

* id
* visible
* eyebrow
* title
* text/description
* CTA text
* CTA link
* background image URL/storage path
* optional mobile background image URL/storage path if reasonable
* background overlay opacity
* image fit: `"cover"` or `"contain"`
* focal point X/Y for cover mode, or at minimum select values like left/center/right and top/center/bottom
* showPosterCards boolean
* optional selected poster IDs or a simple fallback using currently selected `isCollectionBanner` posters if selected IDs are too large for this pass
* optional font overrides
* optional color overrides

Collection banners should be insertable multiple times in the section order. Example:

* Hero
* Collection banner A
* Featured posters
* New arrivals
* Collection banner B
* Explore links
* Brand story
* Value props

7. Standard image specs and crop guidance
   Add a shared image spec helper/config, used by both admin hints and public rendering.

Recommended specs:

* Hero desktop: 1920×600 or wider
* Hero mobile: 1200×900 if mobile-specific image is supported
* Collection banner desktop: 2600×400 or 1920×295, matching 13:2
* Collection banner mobile: 1440×400, matching 18:5, if mobile-specific image is supported
* Add safe-area guidance: keep important text/posters away from edges when using cover mode.

In the admin uploader:

* Show recommended size based on the section.
* After upload, attempt to read natural image dimensions in the browser and show them.
* Show a warning if the uploaded ratio is far from the storefront ratio, e.g. “This image is 1920×600 but the desktop banner is 13:2, so top/bottom cropping may occur in cover mode.”
* Show preview frames for desktop and mobile aspect ratios.
