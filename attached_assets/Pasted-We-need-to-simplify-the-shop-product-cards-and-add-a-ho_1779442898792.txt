Context:

- Repo: https://github.com/johanmarcusholmberg/posterstores

  - hover reveals a selected mockup/lifestyle image

Goal:
Make poster cards visually simpler and more premium, and allow admins to choose which mockup image appears on hover for each poster.

Requested frontend card changes:

1. Simplify PosterCard text content
Update PosterCard so the visible product info below the image only shows:

- category name
- poster name/title
- price line: “From €X”

Remove from the shop card:
- location/city/region line
- sizes line
- “Premium matte print” line
- any extra material/quality row

Keep the product detail page free to show richer information later. The shop grid should stay minimal.

Suggested hierarchy:
CATEGORY NAME — small, muted, uppercase or refined small text
Poster title — clean serif or current title styling, max 2 lines
From €X — clear but not too large

Adjust font sizes/spacing if needed so the card feels elegant and less crowded.

2. Make poster image presentation flatter and more gallery-like
Change the image container styling to look like a clean poster preview, closer to Gallerix-style product grids.

Requested visual direction:
- no rounded image corners
- no obvious app-card rounding
- reduce or remove heavy shadows
- avoid thick frames
- image should feel like a flat poster/product image
- keep a very subtle border/edge only if needed
- maintain consistent aspect ratio
- keep the image area clean and poster-focused

Do not add fake decorative frames around every poster.

3. Add hover image swap to selected mockup
On desktop hover/focus of a product card:
- default state should show the main poster/product image
- hover state should show a selected mockup/lifestyle image for that poster, if one exists
- if no selected mockup exists, keep the main image and do not show a broken/blank state

The transition should be subtle:
- crossfade between main image and hover mockup
- no layout shift
- no aggressive zoom
- avoid flicker when moving mouse

Mobile/touch:
- do not rely on hover
- show the main poster image by default
- do not break tap/click navigation

Accessibility:
- keep card link behavior
- keep keyboard focus state
- hover mockup should not interfere with screen readers
- image alt text should remain sensible

4. Use existing mockup data if available
Inspect current data returned for posters and existing admin/mockup records.

Preferred data model:
- each poster can have one selected hover mockup image
- use an existing “primary”, “featured”, “selected”, or similar mockup field if already available
- if multiple poster mockups exist, use the admin-selected one
- if no selected one exists, optionally fall back to the first primary/featured mockup
- if no mockup exists, show only the main poster image

Do not create duplicate systems if one already exists.
