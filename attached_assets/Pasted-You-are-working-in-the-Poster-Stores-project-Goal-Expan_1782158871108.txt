You are working in the Poster Stores project.

Goal:
Expand the current store admin mockup functionality so the STORE ADMIN owns and manages all poster mockups. The mockup section should support:

1. Managing reusable mockup templates
2. Generating/adding new mockup templates
3. Synchronizing template mockups with posters
4. Automatically compositing/uploading each poster into selected mockup templates with correct fitting, placement, shading, and hover/primary usage
5. Keeping the process suitable for one operator managing many posters

Important workflow context:
- The store/admin should own mockups, not the art style generator.
- The generator may create poster art, but the store should decide which mockups exist, which posters get which mockups, and how mockups are synchronized.
- Do not create a separate “mockup mode” for users. This is an admin/operator workflow.
- Keep changes targeted and avoid unnecessary rebuilds or broad refactors.
- Do not create branches, PRs, or run git automation. Make changes only, then report files changed and ask for approval before any commit/push.

First, inspect the existing codebase before changing anything:
- Current mockup/admin implementation
- Poster image model
- Mockup gallery/product gallery logic
- Existing fields such as isPrimary, isHoverMockup, placement metadata, sourceImageWidth/sourceImageHeight, etc.
- Existing upload/storage APIs
- Existing image generation/upscale/provider code, if any reusable endpoints exist

Then implement the smallest robust version of this feature.

Desired admin UX:

Add or improve a “Mockup Templates” area inside store administration.

Each mockup template should have:
- Name
- Template image
- Optional preview thumbnail
- Store association
- Active/inactive status
- Sort order
- Template category/type, for example:
  - framed wall
  - room scene
  - close-up frame
  - lifestyle
  - product card hover
- Intended use flags:
  - can be primary product mockup
  - can be hover mockup
  - can be gallery mockup
- Poster placement settings:
  - x
  - y
  - width
  - height
  - rotation
  - optional corner/perspective values if the current code can support it safely
- Styling/compositing settings:
  - border radius
  - shadow opacity
  - shadow blur
  - highlight opacity
  - overlay/shading opacity
  - blend mode if already practical
  - fit mode: cover / contain / stretch
- Safe defaults for all settings

Admin should be able to:
- Upload a new mockup template image
- Edit placement visually or numerically
- See a live preview using a selected poster
- Save the template
- Mark template active/inactive
- Set intended use flags
- Select which templates should apply to all posters or selected posters
- Run synchronization

Synchronization behavior:

Add a “Sync mockups” action in admin.

It should support:
- Sync selected poster
- Sync all published posters
- Sync posters missing mockups
- Sync selected templates only
- Dry-run/preview count if easy to add

When sync runs:
- For each selected poster and active selected template:
  - Create or update a generated/composited mockup image
  - Use the poster image as the inserted artwork
  - Apply placement/fitting/shading settings from the template
  - Upload/store the resulting mockup image
  - Connect it to the poster’s mockup/gallery images
  - Respect intended use flags:
    - if template is hover mockup, set resulting image as hover mockup when appropriate
    - if template is primary mockup, allow it to be primary only if no existing primary mockup exists, unless admin explicitly overrides
    - gallery mockups should appear in product gallery
- Avoid duplicate mockups for the same poster/template pair.
- If a poster image changes, the sync should be able to regenerate affected mockups.
- Keep existing manually adjusted mockups safe unless admin explicitly chooses overwrite/regenerate.
