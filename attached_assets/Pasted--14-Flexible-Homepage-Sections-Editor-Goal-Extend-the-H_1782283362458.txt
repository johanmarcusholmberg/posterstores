#14 — Flexible Homepage Sections Editor

## Goal

Extend the Homepage Editor and public `Home.tsx` rendering so every homepage section except the footer can be reordered, shown/hidden, and individually configured.

This phase must support:

* reorderable homepage sections, including Hero
* multiple collection banner instances
* configurable Hero button destination links
* collection banner image fit: `cover` or `contain`
* cover-mode focal point controls
* backwards compatibility with existing stores using the old single `collectionBanner` shape
* a read-only “Global brand defaults” summary in the Homepage Editor with a link to Store Settings

## Workflow constraints

* Do not create branches.
* Do not commit or push.
* Make code changes only.
* Keep the implementation targeted.
* Avoid broad unrelated refactors.
* Preserve existing homepage behavior when no new homepage section config exists.
* At the end, report changed files, what was implemented, what was deferred, and typecheck/test status.

## Done looks like

1. Admin Homepage Editor shows all homepage sections except footer as reorderable rows:

   * Hero
   * Featured posters
   * Collection banner(s)
   * Explore links
   * New arrivals
   * Brand story
   * Value props

2. Each section row has:

   * up/down arrows
   * show/hide toggle
   * clear label
   * normalized ordering

3. Multiple collection banners can be:

   * added
   * duplicated
   * removed
   * shown/hidden
   * reordered like other homepage sections

4. Each collection banner has its own expandable editor with:

   * eyebrow
   * title
   * text/description
   * CTA text
   * CTA link
   * background image URL/storage path
   * overlay opacity
   * image fit: `cover` or `contain`
   * focal point controls for cover mode: horizontal `left | center | right`, vertical `top | center | bottom`
   * `showPosterCards` toggle
   * placeholder note for future section-level font/color overrides

5. Hero section editor keeps existing controls and adds:

   * Primary button link
   * Secondary button link

6. Public homepage uses Hero button links:

   * blank link defaults to `/shop`
   * relative links like `/shop`, `/shop?category=Coastal+Posters`, `/about` must respect the current route prefix behavior
   * absolute `http://` or `https://` URLs should work safely without being prefixed

7. Collection banner rendering:

   * `imageFit: "cover"` uses `object-cover` and focal point via `object-position`
   * `imageFit: "contain"` uses `object-contain` with a suitable background color
   * poster card strip only renders when `showPosterCards` is true
   * existing `isCollectionBanner` poster flag remains the data source for preview poster cards in this phase

8. Image uploaders show recommended dimensions:

   * Hero: `1920×600 recommended`
   * Collection banner: `1920×295 or wider recommended (13:2 ratio)`
   * Mention that cover mode may crop edges and contain mode keeps the full image visible

9. Public `Home.tsx` renders sections according to `homepageVisualConfig.sections` sorted by `sortOrder`.

10. When `sections` is absent, the original fixed order is used as fallback with no visual regression.

11. Existing stores with only old `homepageVisualConfig.collectionBanner` continue to render correctly. Treat the old field as the first collection banner if `collectionBanners` is absent.

12. Homepage Editor shows a “Global brand defaults” read-only summary card:

* current theme colors
* current typography defaults
* link/button to the existing Store Settings route
* do not duplicate the full Store Settings editor in this phase

13. Backend PUT endpoint accepts and validates the new shape with Zod, while still accepting the old shape.

14. Typecheck passes with no new errors.

## Out of scope for this phase

* Applying section-level font/color overrides in public rendering
* Full UI for section-level font/color overrides
* Mobile-specific background images
* Per-banner selected poster IDs
* Full embedding of Store Settings editors inside Homepage Editor
* Browser image dimension auto-detection after upload
* Aspect-ratio mismatch warning after upload

Add type fields for future section-level overrides, but keep the admin UI as a placeholder note and do not apply those overrides in `Home.tsx` yet.

### 6. Hero button link wiring

Create a helper for homepage links, for example:

```ts
function resolveHomepageLink(routePrefix: string | null, href?: string | null): string {
  if (!href || !href.trim()) return makeShopUrl(routePrefix);
  const trimmed = href.trim();
  if (/^https?:\/\//i.test(trimmed)) return trimmed;
  if (trimmed.startsWith("/")) {
    return routePrefix ? `/${routePrefix}${trimmed}` : trimmed;
  }
  return routePrefix ? `/${routePrefix}/${trimmed}` : `/${trimmed}`;
}
```

Use it for:

* Hero primary button
* Hero secondary button
* Collection banner CTA where appropriate

For external URLs, use a safe `<a href>` rather than Wouter `<Link>` if needed. For internal URLs, use existing `<Link>`.

### 7. Collection banner image rendering

* `bottom` → `100%`

Contain mode:

* `object-contain`
* keep full image visible
* use suitable background color from theme/sand fallback

Use existing aspect ratio behavior:

* desktop `13:2`
* mobile `18:5`

Only show poster cards when:

* `banner.showPosterCards === true`

Keep existing `isCollectionBanner` poster selection fallback.
