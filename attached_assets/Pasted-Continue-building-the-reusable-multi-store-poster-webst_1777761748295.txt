WHY THIS MATTERS:
A poster store needs exact purchasable variants.
A user should not just buy “a poster”; they should buy:
- poster
- selected size
- exact price
- currency
- quantity

Orders must later store a snapshot of the selected title, size and price.

DATA MODEL

Create or update poster size/variant model:

poster_sizes:
- id
- poster_id
- size_label
- width_cm
- height_cm
- price
- currency
- active
- sort_order
- created_at
- updated_at

Rules:
- A poster can have multiple sizes.
- Only active sizes are shown publicly.
- Admin can create inactive sizes.
- Price must be positive for active sizes.
- Currency should default from the active store, for example EUR for PostsofSpain.
- Sizes belong to a poster, and the poster belongs to a store.
- Public queries must never expose sizes for posters outside active store.
