Part 5 — Perspective renderer for angled mockups

Update deterministic rendering so a poster can be adapted to angled surfaces.

Current issue:
x/y/width/height only works for straight-on mockups. For posters lying on a table, leaning against a wall, or photographed at an angle, the poster must be warped into the selected four-corner surface.

Implement server-side perspective-aware rendering in or near:

* mockupCompositor.ts

Requirements:

* If effective surface has valid corners, use them.
* Convert normalized corners to pixel coordinates.
* Warp the poster image into the four-corner quadrilateral.
* Composite the warped poster onto the template background.
* Preserve fit mode:

  * cover
  * contain
  * stretch
* Preserve existing render settings where practical:

  * shadow opacity
  * shadow blur
  * highlight opacity
  * overlay opacity
  * border radius if applicable
* Export final flattened JPEG as before.
* Store in mockupImageUrl as before.
* Public gallery behavior remains unchanged.

Technical guidance:

* Sharp alone may not support true 4-point perspective/homography.
* First inspect dependencies.
* If a safe existing dependency/path exists, use it.
* If not, implement a small internal perspective warp utility:

  * compute homography from poster rectangle to destination quadrilateral
  * inverse-map pixels from destination bounding box to source poster image
  * bilinear sample if practical
  * output transparent PNG of the warped poster layer
  * composite warped layer over the background using Sharp
* Keep output sizes bounded to avoid memory issues.
* If true perspective render cannot be safely completed in this pass, do not fake it silently:

  * fall back to the old rectangle renderer
  * return surfaceWarning: "Perspective surface configured, but renderer fell back to rectangle."
  * report limitation clearly.

Important:

* Best-fit rotated rectangle is acceptable only as a fallback, not as the goal.
* The target is real corner-to-corner perspective mapping.
