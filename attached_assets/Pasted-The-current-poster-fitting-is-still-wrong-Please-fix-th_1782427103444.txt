Important correction:
Do not use `posterSizes` / print size labels as the primary source for artwork aspect ratio.

Reason:
`posterSizes` describe available print/product sizes, not necessarily the actual artwork image ratio. A landscape artwork can still have print sizes like A4, A3, 50x70, etc. That makes the code classify a landscape image as portrait, which is what we are seeing now.

New rule:
Use the actual poster artwork image dimensions/aspect ratio to determine fitting.

Preferred data source:

1. Use existing image metadata from the poster/display image if available, for example:

   * width / height fields
   * sourceImageWidth / sourceImageHeight
   * image metadata already stored for display images/mockups
   * any API field that represents the actual artwork file dimensions

2. If no existing metadata is available:

   * use the image naturalWidth/naturalHeight after load as a fallback.
   * The outer card/media container must remain fixed so page layout does not shift.
   * Only the inner artwork wrapper may update after the ratio is known.
   * Use safe object-contain fallback before ratio is known.

3. Do not rely on print size labels except as a last-resort fallback.

Correct fitting behavior:

* Outer card/media area remains fixed.
* Actual artwork is shown inside an inner wrapper.
* Inner wrapper uses the actual artwork ratio, not the print-size ratio.
* Border/ring is applied to the inner wrapper so it hugs the artwork itself.
* Artwork image inside the wrapper uses object-contain.
* Default actual artwork must never be cropped.
* No object-cover for actual poster artwork.
* object-cover is allowed only for desktop hover mockup/lifestyle overlay.

Correct sizing rules:
Portrait artwork:

* Inner wrapper should be portrait-shaped using actual image ratio.
* It may fill the full media height if it fits.
* It should show the full artwork without cropping.
* Small side gaps are acceptable if required.

Landscape artwork:

* Inner wrapper should be landscape-shaped using actual image ratio.
* It should use the available width and adapt height naturally.
* It should be vertically centered in the fixed media area.
* Background above/below should blend with the card/page background.
* It must not keep a tall portrait border around it.
* It must not crop.

Square artwork:

* Inner wrapper should be square/near-square.
* Centered and balanced.
* No cropping.

Unknown ratio:

* Use safe object-contain fallback.
* Never use object-cover.
* Once actual image dimensions are known, update the inner wrapper ratio.

Important CSS/layout guidance:
Do not size the inner wrapper using `h-full` for all orientations.

Use orientation-aware constraints:

* Portrait: height can be 100%, width auto, max-width 100%.
* Landscape: width can be 100%, height auto, max-height 100%.
* Square: constrained by both width and height.

The inner wrapper must never overflow the outer media container.
