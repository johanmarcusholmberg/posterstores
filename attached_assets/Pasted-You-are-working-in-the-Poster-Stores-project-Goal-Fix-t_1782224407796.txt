You are working in the Poster Stores project.

Goal:
Fix the MockupSurfaceEditor zoom/coordinate mismatch bug.

Problem:
When using the Corner / Poster Surface editor and zooming, the template image and poster surface overlay become misaligned. The selected corner handles and dashed polygon no longer match the visible poster/frame area. The screenshot shows the top-left handle appears on the poster, but the top-right / right edge is visually displaced from the actual image surface.

This is likely a coordinate transform bug:

* Image is rendered/scaled/centered one way.
* Polygon/handles are rendered using another scale or offset.
* Drag calculations may be using viewport coordinates instead of the actual transformed image coordinates.
* Zoom/pan/fit likely do not share one single transform model.

Expected architecture:
The editor must use one single coordinate system:

1. Image space

   * Native template image pixels.
   * Example: x=668, y=311.
   * This is the source of truth for editing.

2. Screen/view space

   * The visible editor viewport after fit/zoom/pan.
   * Used only for rendering and pointer events.

Implement or fix explicit transform helpers:

* getEditorTransform()
* imageToScreen(point)
* screenToImage(point)
* normalizedToImage(point)
* imageToNormalized(point)

Rules:

* Image, SVG polygon, corner handles, and drag hitboxes must all use the same transform.
* Dragging a handle must convert pointer coordinates through screenToImage().
* Rendering a handle must use imageToScreen().
* Pixel input fields must read/write image-space pixels.
* Normalized coordinates must be derived from image-space pixels and naturalWidth/naturalHeight.
* Clamp image-space coordinates to [0, naturalWidth] and [0, naturalHeight].

Recommended implementation:
Use a single transformed layer inside the editor viewport.

Example structure:

* viewport div: position relative; overflow hidden
* content layer: position absolute; transform translate(imageLeft, imageTop) scale(scale); transform-origin top left
* inside content layer:

  * img with width=naturalWidth and height=naturalHeight
  * svg overlay with width=naturalWidth and height=naturalHeight, viewBox="0 0 naturalWidth naturalHeight"
  * polygon points in image pixels
  * handle positions in image pixels

This avoids separate scaling math for image and overlay.

Zoom behavior:

* Fit = compute fitScale from viewport size and natural image size.
* 100% = scale 1.0.
* 200% = scale 2.0.
* 400% = scale 4.0.
* Displayed percent should reflect the actual scale being used.
* When zoom changes, image and overlay must remain locked together.

Pan behavior:

* Pan should move the same content layer that contains both image and overlay.
* Alt+drag or middle mouse pan should translate the whole content layer.
* Pan should not alter stored corner coordinates.

Pointer/drag behavior:

* Use pointer capture if possible.
* On pointer down on a corner handle, select that corner.
* On pointer move, convert pointer clientX/clientY to image-space coordinate using the current transform.
* Update only that selected corner.
* Do not use raw clientX/clientY directly as image coordinates.
* Do not use percentage of viewport as image coordinates.

Keyboard nudge:
