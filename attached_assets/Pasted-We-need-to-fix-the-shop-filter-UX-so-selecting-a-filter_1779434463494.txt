- Repo: https://github.com/johanmarcusholmberg/posterstores
- Active shop: PostersOfSpain

- Desired behavior: selecting filters should update the visible products and URL/search params if needed, but the viewport scroll position should remain stable.

Please inspect:
- artifacts/poster-store/src/pages/Shop.tsx
- filter click handlers / sidebar filter links

Goal:
When a user selects or clears a filter on /shop, the page should not automatically scroll to the top. The user's scroll position should stay where it is, unless they explicitly navigate to a different page/product.

Requested changes:

Find why selecting a filter causes the page to jump to the top.

Likely causes to check:
- using normal anchor links instead of buttons
- calling navigate/setLocation in a way that triggers scroll reset
- a ScrollToTop component listening to query-param changes
- remounting the whole Shop page on query changes
- manually calling window.scrollTo
- focus handling that scrolls the page

Update the filter logic so changing filters preserves the current scroll position.

2. Keep URL/query param behavior if currently used
If filters are represented in the URL, keep that behavior.

But update the navigation method so:
- query params change
- browser history still works as before
- the viewport does not reset to top

Use the safest existing project pattern. If needed, use history.replaceState/pushState carefully, but prefer existing router utilities if they support preserving scroll.

3. Do not break real navigation
Only preserve scroll for in-page shop filter/sort/search changes.

Do NOT prevent normal top/navigation behavior when:
- navigating to a product detail page
- navigating to another main page
- using browser back/forward if existing behavior is expected
- loading /shop directly from another route

4. Apply to all shop controls that behave like filters
Verify this for:
- region filter
- category/style filter
