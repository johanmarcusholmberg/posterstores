building the poster webstore project.

STEP 1 GOAL:
Create a reusable multi-store architecture/config layer before building the homepage.

Important:
Do not build this as a hard-coded PostsofSpain-only website.

Build it as a reusable multi-store poster webstore platform where PostsofSpain is the first active storefront.

Future storefronts should be possible, for example:
- PostsofSpain
- PostsofSweden
- PostsofItaly
- PostsofPortugal

All stores should reuse the same:
- Homepage layout
- Shop page
- Poster detail page
- Product cards
- Filters
- Favorites
- Cart
- Checkout/order flow
- Newsletter signup
- Admin poster upload
- Mockup template system

Only store-specific content should change:
- Store name
- Theme colors
- Homepage text
- Country focus
- Regions
- Cities
- Categories
- Tags
- Currency
- Language
- SEO metadata
- Footer links
- Newsletter copy

Create a storefront configuration layer.

Create a file/module such as:

src/config/storefronts.ts

Create an active store resolver.

For MVP:
- activeStoreKey should default to postsofspain.

Prepare the resolver so it can later support:
- domain-based resolution, for example postsofspain.com or postsofsweden.com
- route-based resolution, for example /spain or /sweden

Create a StorefrontProvider or equivalent context/hook:

- getActiveStore()
- useStorefront()
- activeStore.theme
- activeStore.homepage
- activeStore.taxonomy
