Add real store logo upload/storage support and improve the homepage hero responsiveness.

Context:
The Poster Stores app currently uses a text logotype in the header, for example “PostersOfSpain”. I want each store to be able to upload and use a real logo image from the admin area. This must not be just a logo URL field. The uploaded logo should be stored properly and persisted for the selected store.

Also improve the homepage hero layout. On laptop, the hero takes too much vertical space: the heading is too large, appears too far down, and the image collage dominates the viewport. On mobile, the homepage is not working well because “Posters inspired by Spain” appears above the example images in a tall stacked layout, which makes the start page feel too large and not shop-focused enough.

- Do not build a full media library.

Part 1 — Investigate existing upload/storage patterns:
- Check whether the repo already uses Supabase Storage, uploaded poster assets, mockup assets, or any existing file upload endpoints.

Part 3 — Backend upload API:
- Add a protected admin endpoint for uploading/replacing a store logo for a given storeKey.
- Endpoint must require the same admin auth/protection as the other admin store endpoints.
- Validate:
  - storeKey exists
  - file is present
  - file type is png, jpg/jpeg, or webp
  - max file size is reasonable, for example 2MB
- Do not allow unsafe file types.
- Store the file under a clean deterministic path, for example:
  store-assets/{storeKey}/logo/{timestamp-or-hash}-{safe-file-name}
- Return the public logo URL and storage path.
- Save the logo URL/path on the store record.
- If replacing a logo, try to delete the previous stored logo if safe and supported.
- Add either:
  - a remove logo endpoint, or
  - support for clearing logoUrl/logoPath from the store edit page.
- Ensure upload errors are returned clearly to the admin UI.

- Add:

  - upload button/input
  - replace logo action
  - remove logo action

- Show clear loading/error states.
- Keep the UI simple and consistent with the existing admin components.
