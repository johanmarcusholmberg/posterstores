## Issues to fix

### 1. Up/down arrows do not reorder sections

In `AdminHomepageEditor.tsx`, investigate the section move handlers.

Expected behavior:

* Clicking up/down immediately changes the visible row order.
* The updated order is saved into `config.sections`.
* Reordering should be based on the current displayed sorted order, not stale array order.
* After each move, normalize `sortOrder` values to stable increments, e.g. 10, 20, 30.
* Disable up arrow for the first visible row and down arrow for the last visible row.
* Do not disable working arrows incorrectly.

Implementation suggestion:

* Create a helper like:

```ts id="9s4lm3"
function normalizeSectionOrder(sections: HomepageSectionConfig[]): HomepageSectionConfig[] {
  return [...sections]
    .sort((a, b) => a.sortOrder - b.sortOrder)
    .map((section, index) => ({
      ...section,
      sortOrder: (index + 1) * 10,
    }));
}

function moveSection(sectionId: string, direction: "up" | "down") {
  setConfig(prev => {
    const sections = normalizeSectionOrder(getSections(prev));
    const index = sections.findIndex(s => s.id === sectionId);
    if (index < 0) return prev;

    const target = direction === "up" ? index - 1 : index + 1;
    if (target < 0 || target >= sections.length) return prev;

    const next = [...sections];
    [next[index], next[target]] = [next[target], next[index]];

    return {
      ...prev,
      sections: normalizeSectionOrder(next),
    };
  });
}
```

Adapt to the actual helper names in the file.

Also update the helper text. The UI currently says “Drag sections up/down,” but the UI uses arrows. Change it to something like:

> Use the arrows to reorder sections. Toggle the eye icon to show or hide a section.

Do not add drag-and-drop in this phase.

### 2. Make collection banners identifiable in Homepage layout

Currently multiple rows show only “Collection banner,” which makes it impossible to know which one is being moved.

Expected behavior:

* The layout row should display a useful label:

  * `Collection banner 1 · Mediterranean Walls`
  * `Collection banner 2 · Coastal Sale`
  * `Collection banner 3 · Untitled`
* The number should be based on the order of collection banners or their section order.
* If a banner has a title, show it.
* If no title exists, show `Untitled`.

Implementation:

* When rendering layout rows, resolve the section’s `bannerId` to the matching collection banner from `config.collectionBanners`.
* Build a label from collection banner index + title.
* Use the same label in the collection banner editor card header, so the admin can connect the editor card to the layout row.

Also make sure duplicate collection banners receive a unique title fallback, for example:

* `Collection banner 2`
* or copy title + “copy”

### 3. Hero section: add optional extra buttons

The Hero section currently has primary and secondary buttons. Keep those fields for backwards compatibility, but add a user-friendly way to add more buttons.

Add to `HeroVisualConfig`:

```ts id="i81ndh"
export interface HeroButtonConfig {
  id: string;
  label: string;
  link: string;
  variant?: "filled" | "outline";
  visible?: boolean;
}

export interface HeroVisualConfig {
  backgroundImageUrl?: string | null;
  backgroundStoragePath?: string | null;
  backgroundOverlayOpacity?: number;
  primaryButtonText?: string | null;
  primaryButtonVariant?: "filled" | "outline";
  primaryButtonLink?: string | null;
  secondaryButtonText?: string | null;
  secondaryButtonVariant?: "filled" | "outline";
  secondaryButtonLink?: string | null;

  /**
   * Optional extra hero buttons shown after the primary/secondary buttons.
   * Existing primary/secondary fields remain the main backwards-compatible buttons.
   */
  extraButtons?: HeroButtonConfig[];
}
```

Mirror this type in:

* `artifacts/poster-store/src/config/storefronts.ts`

### 4. Make reset safe

The current “Reset to default order” behavior is dangerous if it removes added collection banner sections.

Change behavior and label.

Preferred behavior:

* Rename button to: `Restore default order`
* Clicking it should:

  * restore the built-in section order: Hero, Featured posters, Collection banner/default, Explore links, New arrivals, Brand story, Value props
  * preserve all custom collection banners and their configs
  * preserve custom collection banner sections by appending them after the default sections, unless they already map to the default collection banner
  * preserve visibility flags where possible
  * never delete `collectionBanners`
  * never delete collection banner sections
* Show a confirmation dialog before applying:

  * Title: `Restore default section order?`
  * Body: `This will reorder sections but will not delete your collection banners or their content.`
  * Confirm: `Restore order`
  * Cancel: `Cancel`

Do not include a destructive “full reset” in this phase.

Implementation detail:

* The reset should only operate on `sections`.
* It must not mutate or clear `collectionBanners`.
* If custom collection banner sections exist, append them after default sections with normalized sort orders.
* If a custom section is missing from defaults, preserve it at the bottom.
