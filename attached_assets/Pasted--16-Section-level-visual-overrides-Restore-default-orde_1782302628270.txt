## Part A — Fix Restore default order

### Current bug

`Restore default order` only reverts the latest change. It should always deterministically restore the built-in default section order.

### Expected behavior

When the user clicks `Restore default order` and confirms:

1. Rebuild `config.sections` from `DEFAULT_HOMEPAGE_SECTIONS`.
2. Preserve existing visibility flags where possible.
3. Preserve existing section visual overrides where possible.
4. Preserve all custom collection banner sections.
5. Append custom collection banner sections after the default sections.
6. Never delete or mutate `config.collectionBanners`.
7. Never delete banner content, images, CTA text, image fit, focal point, poster-card toggle, or visual overrides.
8. Normalize `sortOrder` to stable increments: 10, 20, 30...

### Implementation guidance

Use a deterministic helper based on the default constant, not previous UI state history.

Example shape:

```ts id="fyauyo"
function restoreDefaultSectionOrder(prev: HomepageVisualConfig): HomepageVisualConfig {
  const currentSections = normalizeSectionOrder(getSections(prev));
  const currentById = new Map(currentSections.map((s) => [s.id, s]));

  const defaultSections = DEFAULT_HOMEPAGE_SECTIONS.map((defaultSection) => {
    const existing = currentById.get(defaultSection.id);
    return {
      ...defaultSection,
      visible: existing?.visible ?? defaultSection.visible,
      fontOverrides: existing?.fontOverrides ?? defaultSection.fontOverrides ?? null,
      colorOverrides: existing?.colorOverrides ?? defaultSection.colorOverrides ?? null,
    };
  });

  const defaultIds = new Set(DEFAULT_HOMEPAGE_SECTIONS.map((s) => s.id));

  const customSections = currentSections.filter((section) => {
    if (defaultIds.has(section.id)) return false;
    return true;
  });

  return {
    ...prev,
    sections: normalizeSectionOrder([...defaultSections, ...customSections]),
  };
}
```

Adapt to the real helper names in `AdminHomepageEditor.tsx`.

Important:

* The reset must not simply undo the last move.
* The reset must not clear `collectionBanners`.
* The reset must not remove extra collection banner section rows.
* Cancel should close confirmation without changing state.
