“Save failed”
“Internal Server Error”

The failing area is the “Poster placement area” / placement rectangle for mockup templates. The current UI stores/edits placement as percentages, e.g. left/top/width/height, but I also want pixel-level precision for accurate placement.

3. Keep backward compatibility with existing percentage-based placement data.
4. Make the editor easier to use accurately.

2. Support both percentage and pixel placement
Currently placement appears to be percentage-based. Add support for pixel-accurate editing while preserving the existing normalized percentage placement.

Recommended model:
- Continue storing canonical placement as normalized percentages for responsive mockup rendering:
  - placementLeftPct
  - placementTopPct
  - placementWidthPct
  - placementHeightPct
- Also allow the admin editor to show and edit pixel values based on the actual natural dimensions of the background image:
  - placementLeftPx
  - placementTopPx
  - placementWidthPx
  - placementHeightPx

If the database already has a JSON placement object, extend it safely rather than breaking existing data. For example:
{
  "x": 36,
  "y": 10,
  "width": 22,
  "height": 35,
  "unit": "percent",
  "sourceImageWidth": 1200,
  "sourceImageHeight": 1600
}

Or, if current code already has another naming convention, keep that convention and extend it consistently.

The important behavior:
- Percentages remain the canonical saved values used for rendering.
- Pixel fields are derived from the image’s natural width/height.
- When the user changes pixel values, update the corresponding percentages.
- When the user changes percentages, update the corresponding pixel values.
- Store image dimensions with the placement metadata when available, so future edits can accurately convert between px and %.

Conversion:
leftPct = leftPx / imageNaturalWidth * 100
topPct = topPx / imageNaturalHeight * 100
widthPct = widthPx / imageNaturalWidth * 100
heightPct = heightPx / imageNaturalHeight * 100

And reverse:
leftPx = round(leftPct / 100 * imageNaturalWidth)
topPx = round(topPct / 100 * imageNaturalHeight)
widthPx = round(widthPct / 100 * imageNaturalWidth)
heightPx = round(heightPct / 100 * imageNaturalHeight)

5. Backend/API validation

Validation rules:
- left/top/x/y must be >= 0
- width/height must be > 0

- reject NaN, null, undefined, empty strings, and non-numeric values
