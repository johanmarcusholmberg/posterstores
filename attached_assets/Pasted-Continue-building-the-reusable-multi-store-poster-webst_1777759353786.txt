ASSET PRINCIPLE:
Even though the current schema only has one imageUrl field, structure the mockup system with this distinction in mind:

1. Master printable image
- The production-quality poster image.
- Used for print/fulfillment later.
- Currently represented by imageUrl until separate master/preview fields are added.

2. Preview image
- Used in store cards and product galleries.
- Currently also represented by imageUrl.

3. Mockup images
- Presentation assets only.
- Used to show poster in frames/interiors/lifestyle scenes.
- Must not replace the printable production image.

- id
- poster_id
- mockup_template_id nullable
- mockup_image_url nullable
- sort_order integer
- is_primary boolean
- created_at
- updated_at

Rules:
- mockup_templates.store_key can be null for global templates.
- If store_key is null, the template can be used by all stores.
- If store_key is set, the template is only available for that store.
- poster_mockups belong to a poster.
- A poster can have multiple poster_mockups.
- Only one poster_mockup should be primary per poster.
- If one mockup is marked primary, all other mockups for that poster should be set to non-primary.
- Poster mockup mutations must verify that the poster belongs to the requested storeKey.
